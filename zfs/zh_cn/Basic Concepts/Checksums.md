# Checksums

## 校验和(Checksum)及其在ZFS中的应用

端到端校验和是ZFS的核心特性，也是ZFS区别于其他RAID实现和文件系统的重要优势。

端到端校验和的优势包括：

- 从存储介质读取时检测数据损坏
- 检测到损坏的块将自动修复（若可行），通过合理配置的存储池中的RAID保护机制，或冗余副本实现（参见zfs的``copies``属性）
- 定期扫描可检测潜在介质劣化（位腐蚀）及其他来源的损坏并修复
- ZFS复制流（`zfs send`与`zfs receive`）的校验确保接收数据未受存储/传输机制损坏

## 校验算法

ZFS中的校验算法可针对数据集（文件系统或卷）进行变更。
  每个数据块使用的校验算法存储于块指针（元数据）中。
  块校验值在写入时计算生成，因此变更算法仅影响变更后发生的写入操作。

通过设置``checksum``属性可更改数据集的校验算法：

```bash
zfs set checksum=sha256 pool_name/dataset_name
```

| **校验和** | **支持数据去重功能(dedup)?** | **兼容其他ZFS实现?**                 | **备注**                                                                        |
| ---------- | ---------------------------- | ------------------------------------ | ------------------------------------------------------------------------------- |
| on         | 参见注意事项                 | 兼容                                 | ``on`` 是未去重数据集的 ``fletcher4`` 缩写，而去重数据集的缩写则是 ``sha256``。 |
| off        | 不允许                       | 兼容                                 | 不要用这个                                                                      |
| fletcher2  | 不允许                       | 兼容                                 | 已弃用的 Fletcher 校验和实现，改用 ``fletcher4`` 代替                           |
| fletcher4  | 不允许                       | 兼容                                 | Fletcher算法，也用于``zfs send`` 数据流传输                                     |
| sha256     | 允许                         | 兼容                                 | 去重后数据集的缺省值                                                            |
| noparity   | 不允许                       | 兼容                                 | 不要用这个                                                                      |
| sha512     | 允许                         | 需要存储池的`org.illumos:sha512`功能 | 加盐的`sha512`, 目前在boot池上的任何文件系统都不支持                            |
| skein      | 允许                         | 需要存储池的`org.illumos:skein`功能  | 加盐的`skein`, 目前在boot池上的任何文件系统都不支持                             |
| edonr      | 参见注意事项                 | 需要存储池的`org.illumos:edonr`功能  | 加盐的`edonr`, 目前在boot池上的任何文件系统都不支持;出于谨慎考虑                |
| blake3     | 允许                         | 需要存储池的`org.openzfs:blake3`功能 | 加盐的`blake3`, 目前在boot池上的任何文件系统都不支持                            |

注意事项: 出于谨慎考虑, 当Edon-R哈希算法与数据去重一起使用时, 需要进行验证, 因此它会自动启用`verify`

## 校验和加速

ZFS 可以把校验运算(checksum operations)卸载到 Intel QuickAssist Technology (QAT) 适配器上执行

## 校验和微基准测试

当`zfs.ko`内核模块加载时, 某些 ZFS 功能会运行微基准测试(micro benchmarks), 以确定校验的最佳算法。
    测试结果可以在`/proc/spl/kstat/zfs`目录中查看。
    获胜的算法会被标记为`fastest`, 并成为缺省算法。
    缺省值可以通过设置 ZFS 模块参数来覆盖

| **校验和**    | **测试结果**                         | **ZFS模块参数**                                   |
| ------------- | ------------------------------------ | ------------------------------------------------- |
| **Fletcher4** | /proc/spl/kstat/zfs/fletcher_4_bench | zfs_fletcher_4_impl                               |
| **其它**      | /proc/spl/kstat/zfs/chksum_bench     | zfs_blake3_impl, zfs_sha256_impl, zfs_sha512_impl |
