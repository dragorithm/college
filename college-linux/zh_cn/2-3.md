## 先决条件

### 环境变量

```bash
su

# Basic

```

关于 `bin` 概念的补充说明：

- **bin**：普通用户可访问的基本命令
- **sbin**：面向管理员的系统管理命令

### 创建分区

```bash
sudo apt install gdisk
sudo sgdisk -n 0:0:0 /dev/sda
```

### 使用 mkfs 命令建立文件系统

```bash
sudo mkfs.vfat -F 32 /dev/sda1
```

### 使用 fsck 命令检查文件系统

```bash
sudo fsck.vfat /dev/sda1
```

### 删除硬盘分区

```bash
sudo sgdisk -d 1 /dev/sda
sudo sgdisk --label-clear /dev/sda
```

## 其他硬盘管理工具

使用其他磁盘管理工具（例如 `dd`、`df`、`du`、`mount`）进行磁盘优化和配置。

### dd

```bash
sudo dd if=/dev/zero of=/dev/sda bs=512 count=2048
dd if=/dev/zero of=disk.img bs=1M count=100
```

- **命令 1**：向 /dev/sda 写入 2048 次 0，每次写入 512 字节
- **命令 2**：创建一个虚拟磁盘映像文件

### du

```bash
du /root
du -sh /home
sudo du -sh /* | sort -h | tail -n 10
```

参数说明：

- `-s`: 统计
- `-h`: 人类可读的输出（GB/MB）

### mount

```bash
mkfs.ext4 disk.img
sudo mkdir /mnt/vdisk
sudo mount disk.img /mnt/vdisk
```

### df

```bash
df -h
df -h /mnt/vdisk
```

## RAID

```bash
su
apt install -y zfsutils-linux zfs-dkms linux-headers-amd64
```

### 内核

*注意*：重启时，请选择支持 ZFS 的内核映像。

```bash
root@debian:~# modprobe zfs
modprobe: FATAL: Module zfs not found in directory /lib/modules/6.12.74+deb13+1-amd64
```

上文显示的 `kernel 6.12.74+deb13+1-amd64` 是您在启动时**不应**选择的内核。

### raidz1

```bash
zpool create -f -o ashift=12 tank raidz1 /dev/sda /dev/sdb /dev/sdc /dev/sdd
```

### zvol

```bash
zfs create -V 5G tank/xfs
apt install -y xfsprogs
mkfs.xfs /dev/zvol/tank/xfs
```

### 查看 raid

```bash
zfs list
```

### 挂载点

```bash
zfs set mountpoint=/mnt/tank tank
```

### 数据恢复

```bash
zpool offline tank /dev/sdc
zpool status tank
zpool replace tank /dev/sdc /dev/nvme0n2
zpool status tank
```

**说明**：将磁盘设为离线状态以模拟磁盘故障，然后用新磁盘替换它——RAID 将自动开始重建。

### 停用 raid

```bash
zpool export tank
```
