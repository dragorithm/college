# git commit

提交信息应使用英文书写，以确保一致性和可读性。

## 规则

\[类型\](文件路径 1, 文件路径 2, ..., 文件路径 n): 消息  
描述

脚注

## 描述

必填

- [类型]
- (文件路径 1, 文件路径 2, ..., 文件路径 n)
- : 消息

## 类型

- **feat**: 新功能
- **fix**: 错误修复
- **docs**: 更新 ./docs 下的文档
- **style**: 格式调整，不影响逻辑
- **refactor**: 代码重写 无新增功能 无错误修复
- **perf**: 性能优化
- **revert**: 撤销提交
- **chore**: 其他
