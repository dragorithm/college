# git commit

提交信息应使用英文书写，以确保一致性和可读性。

如果git文档更新, 提交使用新的规范, 包括git文档本身

## 规则

\[类型\](文件路径 1, 文件路径 2, ..., 文件路径 n): 消息  
描述

脚注

## 描述

必填

- [类型]
- (文件路径 1, 文件路径 2, ..., 文件路径 n)
- : 消息

## 类型

- **feat**: 功能
  - add: 引入新功能；可能涉及添加新模块或在现有模块中添加新内容
  - del: 从模块中移除内容或删除整个模块
  - up: 更新现有功能
- **fix**: 缺陷修复
  - security: 修复与安全相关的问题
  - defect: 修复非安全性的缺陷
- **docs**: 更新 ./docs 下的文档
  - git: 更新与git相关的
  - '': 其它, 不需要子type
- **style**: 格式调整，不影响逻辑
- **refactor**: 代码重写, 无新增功能, 无错误修复
- **perf**: 性能优化
- **revert**: 撤销提交
- **chore**: 其他

## 三层提交结构

- 类型：描述了做了什么。
- 范围：描述了更改的位置。
- 消息：应重点说明为何进行更改；仅在必要时添加具体细节。
    提交消息时，所有文字应使用小写，单词之间用空格分隔，且结尾处不加句号。
