# 重写提交

以下是 Git 中涉及重写历史的常见操作。

## 注意事项

重写历史会导致提交哈希值（SHA-1/SHA-256）发生变化。

如果历史已推送到远程分支，重写时需要使用以下命令：

```bash
git push --force
```

或者，更安全的方法是：

```bash
git push --force-with-lease
```

**注意**：强制推送可能会引发问题。
    如果其他开发者已推送了新提交，而你在未先拉取的情况下重写历史，
    你的强制推送可能会覆盖远程分支上的最新历史。
