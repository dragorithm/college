# 用户

## 常用键

- **name**：提交时显示的用户名
- **email**：提交时显示的电子邮件地址

这两个键是必填的，因为它们用于标识提交记录的作者和提交者。

## 其他键

在多数情况下较少使用，但偶尔会派上用场：

- **useConfigOnly**：若设置为 `true`，Git 将仅使用配置文件中的用户信息，而不会尝试从系统环境变量中推断用户信息
- **signingKey**：指定用于签署提交的 GPG 密钥 ID

## 提示

- 对于单个用户，**用户级(global)**配置已足够。
- 对于多用户环境，可使用 **仓库级(local)**配置来区分身份。

常见的多用户场景示例：

- 多名开发者共用同一台设备，需要区分提交记录
- 一台设备同时用于工作和学校项目，避免身份混淆
- 使用特定邮箱地址（而非个人邮箱）向开源项目提交代码
